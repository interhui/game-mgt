# Worklog: 菜单和工具条优化

## 问题描述
用户需求：对菜单项和工具条进行优化

## 需求分析

### 1. 菜单优化
- 将"文件"菜单中的"刷新游戏库"、"设置"、"退出"转移到"游戏"菜单
- 在"游戏"菜单中增加"添加游戏"子菜单，包含：手动添加、IGDB导入、JSON导入
- "视图"菜单项以中文方式展示

### 2. 工具条优化
- 移除"添加游戏"按钮
- 移除"导入JSON"按钮
- 在"切换视图"和"设置"按钮之间增加"刷新游戏库"按钮

## 实现步骤

### 1. 修改 main.js - 更新菜单结构

#### "文件"菜单 → "游戏"菜单
**修改前**：存在"文件"菜单和"游戏"菜单
**修改后**：
- "文件"菜单被删除
- "游戏"菜单包含：刷新游戏库、添加游戏（子菜单）、设置、退出

#### "视图"菜单中文化


**修改后**：

### 3. 修改 preload.js - 新增 IPC 事件监听

新增：

## Bug 修复

### Bug 1: 刷新游戏库图标位置不正确
**问题**: 刷新游戏库图标应该在"切换视图"和"设置"之间
**修复**: 调整 toolbar-right 中的按钮顺序
- 修改前: `[batch-add] [search] [refresh] [view-toggle] [settings]`
- 修改后: `[batch-add] [search] [view-toggle] [refresh] [settings]`

### Bug 2: 添加游戏时出现 NullPointerException
**问题**: `main.js:994 Uncaught (in promise) TypeError: Cannot read properties of null (reading 'addEventListener')`
**原因**: 移除了工具条上的"添加游戏"按钮，但未移除对应的 JavaScript 事件绑定
**修复**:
- 移除 `elements.addGameBtn` 元素引用
- 移除 `elements.addGameBtn.addEventListener(...)` 事件监听
- 移除 `elements.importJsonBtn` 元素引用
- 移除 `elements.importJsonBtn.addEventListener(...)` 事件监听
